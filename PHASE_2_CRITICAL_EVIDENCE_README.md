## Technical Requirements

### Log Format Requirements
All system logs must include:
- Date/time stamps (YYYY-MM-DD HH:MM:SS SAST)
- Administrator identity (who made changes)
- Target user (who was affected)
- Specific changes (before/after permissions)
- Session IDs and IP addresses
- Business justification fields (if available)

### Authentication Requirements
All evidence must be:
- Certified by service provider as authentic
- Provided in original digital format
- Include chain of custody documentation
- Accompanied by provider affidavit if possible
- Verified for data integrity (checksums)

### Date Range Requirements
- **Primary Period:** June 7, 2025 - October 30, 2025 (critical events)
- **Extended Period:** January 1, 2024 - October 30, 2025 (pattern establishment)
- **Historical Period:** 2015 - 2025 (collaborative model evidence)
