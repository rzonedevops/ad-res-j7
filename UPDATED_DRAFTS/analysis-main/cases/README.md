# Cases Directory

This directory contains case management files organized by status and type.

## Directory Structure

```
cases/
├── active/     # Currently active cases
├── closed/     # Completed and closed cases
├── templates/  # Case templates for new cases
└── README.md   # This file
```

## Organization Rules

- **Active Cases**: Place ongoing case files in `active/` subdirectory
- **Closed Cases**: Archive completed cases in `closed/` subdirectory  
- **Templates**: Store case templates and boilerplate files in `templates/`
- **Naming Convention**: Use format `case_YYYY_XXXXXX` for case directories
- **Documentation**: Each case should include a README.md with case overview

## Usage

### Creating a New Case
1. Copy appropriate template from `templates/` directory
2. Create new case directory in `active/` using naming convention
3. Update case metadata and documentation
4. Begin case analysis using framework tools

### Closing a Case
1. Generate final reports and summaries
2. Archive all case materials
3. Move complete case directory from `active/` to `closed/`
4. Update case status in tracking systems

## Templates

Case templates provide standardized starting points for different types of analyses:
- Criminal case timelines
- Civil litigation analysis  
- Corporate investigation
- Evidence management

See `templates/` directory for available templates.
