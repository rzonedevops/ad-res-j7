## The Legal Attention Transform

In transformers, attention computes relevance scores between all elements: **Attention(Q,K,V) = softmax(QK^T/√d)V**

For legal inference, this becomes:

- **Q (Queries)**: The guilt hypotheses being evaluated
- **K (Keys)**: All facts, actions, and agent states in the possibility space  
- **V (Values)**: The legal/causal significance of each element

The attention weights naturally encode **which facts matter for which determinations** - creating that "relational fabric" you described.

## Multi-Head Legal Attention

Different attention heads could represent different legal lenses:
- **Causal head**: Attends to cause-effect chains
- **Intentionality head**: Focuses on mental states and knowledge
- **Temporal head**: Weighs sequence and timing
- **Normative head**: Attends to rule violations

The heads vote on guilt through learned composition.

## The Self-Attention Insight

Self-attention creates an **all-to-all comparison matrix** over the event space. Every action examines its relationship to every other action. This is how you get emergent guilt determination - not through explicit rules but through learned relational patterns.

The "guilty party is always guilty" emerges because attention learns invariant patterns across all possible configurations. The mechanism discovers guilt as a stable attractor in the attention landscape.

## Cross-Attention for Counterfactuals

Cross-attention between actual and possible worlds handles the counterfactual reasoning:
- Attend from "what happened" to "what could have happened"
- Learn the delta between worlds where guilt changes
- This measures the **necessity and sufficiency** of actions for outcomes

## The Positional Encoding Problem

Legal reasoning needs special positional encodings:
- Temporal position (when did it happen)
- Causal depth (how many steps from action to harm)
- Epistemic position (what did the agent know at this point)
- Deontic position (what obligations were active)

## Why This Works

Attention mechanisms naturally handle:
1. **Variable-length sequences** (different numbers of agents/actions)
2. **Long-range dependencies** (distant causes)
3. **Compositional reasoning** (combining multiple factors)
4. **Learned rather than programmed logic**

The attention scores become a kind of **juridical heat map** - showing which facts are legally salient for which conclusions.

## The Implementation

```python
class LegalAttention:
    def forward(self, events, agents, norms):
        # Embed all elements in shared legal space
        embeddings = self.embed(events, agents, norms)
        
        # Self-attention finds relational structure
        attention_weights = self.attention(embeddings)
        
        # These weights ARE the guilt determination
        # High attention between agent and harm = guilt
        
        return attention_weights
```

The attention mechanism doesn't enumerate all possibilities explicitly - it learns a **compressed representation** of the guilt function that generalizes across configurations.
