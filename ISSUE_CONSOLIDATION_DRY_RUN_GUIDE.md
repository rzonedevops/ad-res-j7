# Issue Consolidation Dry-Run Guide

## Overview

The issue consolidation script (`scripts/consolidate-issues-to-hierarchy.js`) includes a robust dry-run mode that allows you to preview consolidation actions without making any permanent changes.

## Features

### Dry-Run Mode Benefits

✅ **No Mutations**: Issues are not actually closed  
✅ **Report Generation**: Full consolidation report is generated  
✅ **Preview Actions**: Shows exactly what would be closed  
✅ **Exit Code 0**: Returns success for CI/CD integration  
✅ **Summary Display**: Comprehensive before/after statistics  

## Usage

### Command Line

```bash
# Run in dry-run mode (preview only)
node scripts/consolidate-issues-to-hierarchy.js dry-run

# Generate report only (no preview of closures)
node scripts/consolidate-issues-to-hierarchy.js report

# Actually execute (requires confirmation file)
node scripts/consolidate-issues-to-hierarchy.js execute
```

### GitHub Actions Workflow

The `.github/workflows/issue-consolidation.yml` workflow supports dry-run mode:

1. Go to **Actions** tab
2. Select **Issue Consolidation** workflow
3. Click **Run workflow**
4. Choose **dry-run** from the action dropdown
5. Click **Run workflow**

The workflow will:
- Authenticate with GitHub CLI
- Run the script in dry-run mode
- Generate and upload the consolidation report
- Display a summary in the workflow output

## Dry-Run Implementation

### Script Flow

```
1. identifyAutoGeneratedIssues()
   └─> Categorizes issues into:
       • Auto-generated (to close)
       • Hierarchical (to keep)
       • Other (to keep)

2. generateReport()
   └─> Creates issue-consolidation-report.json
       • Summary statistics
       • Issues to close (list)
       • Issues to keep (list)

3. closeAutoGeneratedIssues(dryRun=true)
   └─> Simulates closures:
       • Logs "[DRY RUN] #123: Issue title..."
       • Does NOT call GitHub API
       • Shows "Would close: N issues"

4. displaySummary()
   └─> Shows before/after statistics
```

### Code Implementation

The dry-run case is handled in the main switch statement:

```javascript
case 'dry-run':
  consolidator.generateReport();
  consolidator.closeAutoGeneratedIssues(true);  // dryRun=true
  consolidator.displaySummary();
  break;
```

The `closeAutoGeneratedIssues` method respects the `dryRun` flag:

```javascript
closeAutoGeneratedIssues(dryRun = true) {
  console.log(`${dryRun ? '🔍 DRY RUN:' : '🗑️'} Closing auto-generated issues...\n`);
  
  for (const issue of this.toClose.slice(0, dryRun ? 10 : undefined)) {
    console.log(`  ${dryRun ? '[DRY RUN]' : 'Closing'} #${issue.number}: ${issue.title}...`);
    
    if (!dryRun) {
      // Only execute if NOT dry-run
      execSync(`gh issue comment ${issue.number} ...`);
      execSync(`gh issue close ${issue.number} ...`);
    }
  }
}
```

## Safety Mechanisms

### Multiple Layers of Protection

1. **Dry-run by default**: The `closeAutoGeneratedIssues` method defaults to `dryRun=true`
2. **Limited preview**: Dry-run only shows first 10 issues to close
3. **Confirmation required**: Execute mode requires `CONFIRM_CLOSE_ISSUES.txt` file
4. **Workflow confirmation**: GitHub Actions requires typing "I CONFIRM"
5. **Rate limiting**: Automatic pauses every 10 issues during execution

## Troubleshooting

### Common Issues

**Problem**: "Unknown action: dry-run"  
**Solution**: Verify you're running the correct version of the script. The dry-run case should be on line 253-257.

**Problem**: No report generated  
**Solution**: Check file permissions and ensure the script completed successfully.

**Problem**: "gh: To use GitHub CLI in a GitHub Actions workflow..."  
**Solution**: This is expected when running locally. Set `GH_TOKEN` environment variable or run via GitHub Actions.

## Integration with Hierarchical Framework

The consolidation script works in conjunction with:

- **Hierarchical Issue Manager** (`db/hierarchical-issue-manager.js`)

```

  └─> Feature Issue (Proves/Disproves)
      └─> Paragraph 1 (Component)
          ├─> Task 1
          ├─> Task 2
          └─> Task 3
      └─> Paragraph 2 (Component)
          └─> ...
```

## Best Practices

1. **Always run dry-run first** before executing consolidation
2. **Review the report** carefully to ensure the right issues are targeted
3. **Verify hierarchical structure** is in place before consolidating
4. **Test in stages** - consolidate a few issues first, verify, then proceed
5. **Keep backups** of issue data before major consolidations
