## 🛠️ ACTION REQUIRED: Fix 'dry-run' Handling in Issue Consolidation Workflow

### Summary
The Issue Consolidation workflow fails when the 'dry-run' action is selected. The failure is caused by the absence of 'dry-run' handling in the script (`scripts/consolidate-issues-to-hierarchy.js`), resulting in the error:

```
Unknown action: dry-run
```

### Root Cause
- The script is missing a handler for 'dry-run'.
- When the workflow is triggered with action: dry-run, the workflow exits with code 1.

### Solution: Implementation Steps

#### 1. **Update Action Dispatcher**
- **File:** `scripts/consolidate-issues-to-hierarchy.js`
- **Task:** Add a case for 'dry-run' in the action dispatcher.

```js
switch (process.argv[2]) {
  case 'report':
    // existing logic
    break;
  case 'dry-run':
    // Implement dry-run logic: simulate, do not mutate state
    console.log("Running consolidation in dry-run mode. No issues will be closed.");
    require('./db/hierarchical-issue-manager').run({ dryRun: true });
    break;
  case 'execute':
    // execution logic
    break;
  default:
    console.error('Unknown action: ' + process.argv[2]);
    process.exit(1);
}
```
